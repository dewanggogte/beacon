## Task: Set up vitest test framework (BEA-1, critical)

### Approach

Installed vitest at the monorepo root as a devDependency. Created a single `vitest.config.ts` that discovers `*.test.ts` files across all workspace packages via the glob pattern `packages/*/src/**/*.test.ts`. This avoids needing per-package vitest configs while still supporting workspace-scoped tests.

Chose vitest over Jest because:
- Native TypeScript + ESM support (no `ts-jest` or `@swc/jest` needed)
- This is an ESM-only monorepo (`"type": "module"` + `NodeNext` module resolution)
- Vitest is the de facto standard for modern TypeScript projects

## Task: Add unit tests for disqualifier scoring logic (BEA-3, high)

### Approach

Wrote 54 unit tests covering both `checkDisqualifiers()` and `checkHardGates()` — the two gatekeeper functions that determine which companies get disqualified from the scoring pipeline. Used minimal fixture objects with `as any` to avoid pulling in the full `EnrichedSnapshot` type with its 95+ fields.

Tests organized into 4 parts:
1. **Basic disqualifiers** (17 tests): All 8 rule types including sector-based skips
2. **Enriched-path disqualifiers** (6 tests): Net losses and promoter decline
3. **Hard gates** (22 tests): All 9 gates (Piotroski, Altman, ROCE, revenue decline, Beneish, OCF/profit, pledge, completeness, other income)
4. **Edge cases** (9 tests): Null handling, multiple rules, sector matching

### Bug Found

Discovered an inconsistency in sector matching between the two modules:
- `disqualifier.ts` line 88: `sector.includes('finance')` — does NOT match "Financial Services"
- `hard-gates.ts` `isBankingOrNBFC()`: `sector.includes('financial')` — correctly matches

Impact: The D/E > 3 rule fires for "Financial Services" companies in the basic disqualifiers but is correctly skipped in the hard gates. This is a low-severity issue since hard gates run separately and the financial sector detection there is correct. Documented in the test with a comment.
